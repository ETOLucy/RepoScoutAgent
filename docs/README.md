# RepoScoutAgent 文档

[简体中文](README.md) | [English](README.en.md)

本目录记录当前工程计划和可追溯的性能历史。产品功能、安装配置和 API 以仓库根目录的
[中文 README](../README.md) 为准；离线质量评测见 [评测说明](../evals/README.md)。

## 当前功能

- GitHub 与自托管 SearXNG 多源候选召回；没有网页搜索服务时保留 GitHub 路径。
- 宽召回、仓库级重排、L1 文档证据、L2 静态实现迹象和逐项引用校验。
- 核心仓库、移动端同步、对象存储、反向代理等多组件方案及接口兼容性证据。
- 可选 Deep Code 代码职责解释，与 L2 功能存在性验证严格分离。
- 协作式需求确认，支持确认、自然语言修改和跳过。
- SQLite 会话历史、消息、待确认检查点和研究快照持久化。
- 前端结构化执行过程、阶段状态和节点耗时；不保存模型私有思维链。
- 中文或中英混合请求使用中文叙述，纯英文请求使用英文；GitHub 检索词可保持英文。
- 严格搜索默认不静默降级；只有用户主动开启快速降级才使用规则任务契约。

## 维护约定

1. 默认无语言后缀的 Markdown 文档使用简体中文，英文镜像使用 `.en.md`。
2. 新增用户可见功能时，至少同步更新根 README、本文档和相关专项文档。
3. 性能历史不删除旧基线；行为改变时增加“历史说明”或新版本章节，避免把旧配置写成当前配置。
4. 未经重复测量的数据只能写为单次观察，不能表述为 p50、p95 或吞吐量结论。
5. `docs/RESUME.md` 是本地简历材料并由 Git 忽略，不属于公开文档。
